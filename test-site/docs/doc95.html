        Stop Merging Broken Code
        • The 4.3.0 release of Nodepool implements a long-awaited feature: you can now scope resource quotas to separate Zuul tenants, providing native configuration for group-specific resource management.
        • Zuul 4.9.0 and 4.10.0 are now available, adding a new buildset timeline feature in the web dashboard, more administrative tools for scheduler state and secret key management, and metrics for tracking tenant-specific node request volume. Note that Zuul 4.9.0 and later requires extra care at upgrade to load secret keys if the previous version run was less than 4.6.0, and Zuul 4.10.0 requires Nodepool 4.3.0 or later.
        • A new zuul-registry 1.1.0 release introduces support for restricted read authentication, and brings fixes for a variety of subtle content handling errors.
